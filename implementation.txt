RENCANA IMPLEMENTASI PENELITIAN: ANALISIS ROBUSTNESS DETEKSI WAJAH YOLOv8

PHASE 1: PERSIAPAN LINGKUNGAN (ENVIRONMENT SETUP)
Detail Subphase:
1.1 Setup Google Colab:
    - Menyiapkan notebook baru di Google Colab.
    - Menghubungkan Drive untuk penyimpanan permanen dataset dan model (.pt).
    - Memastikan runtime menggunakan GPU (T4 atau versi lebih tinggi).
1.2 Instalasi Library Utama:
    - Install Ultralytics (pip install ultralytics).
    - Pastikan OpenCV dan NumPy sudah tersedia.
1.3 Verifikasi Hardware:
    - Cek ketersediaan GPU menggunakan command `nvidia-smi`.

PHASE 2: AKUISISI DAN PRE-PROCESSING DATA
Detail Subphase:
2.1 Download Dataset Kaggle:
    - Mencari dataset deteksi wajah (contoh: WIDER FACE).
    - Ekspor API Key Kaggle ke Colab untuk download langsung.
2.2 Strukturisasi Dataset:
    - Membagi data menjadi folder: train, val, dan test.
    - Memastikan format anotasi sesuai standar YOLO (txt file per image).
2.3 Exploratory Data Analysis (EDA):
    - Melakukan visualisasi sampel image untuk memastikan label/bounding box sudah benar.

PHASE 3: SKENARIO 1 - PELATIHAN BASELINE (TANPA AUGMENTASI)
Detail Subphase:
3.1 Konfigurasi Model:
    - Memilih varian model (YOLOv8n untuk efisiensi atau YOLOv8s).
    - Menyiapkan file data.yaml yang mengarah ke path dataset.
3.2 Proses Training Baseline:
    - Menjalankan training selama N epoch (misal: 50-100 epoch).
    - Menyimpan weights terbaik (best.pt).
3.3 Evaluasi Baseline:
    - Mendapatkan nilai Precision, Recall, dan mAP pada kondisi pencahayaan normal.
    - Menguji model baseline pada citra kondisi low-light asli untuk melihat penurunan performa.

PHASE 4: PENGEMBANGAN TEKNIK DATA AUGMENTATION LOW-LIGHT
Detail Subphase:
4.1 Implementasi Script Augmentasi (OpenCV):
    - Membuat fungsi Brightness Adjustment: Menurunkan nilai intensitas secara linear.
    - Membuat fungsi Gamma Correction: Menggunakan tabel pencarian (lookup table) untuk transformasi nonlinear.
4.2 Batch Processing:
    - Menerapkan fungsi tersebut ke seluruh dataset training untuk mensimulasikan kondisi low-light.
4.3 Validasi Augmentasi:
    - Memastikan hasil citra yang telah diaugmentasi masih memiliki informasi wajah yang terlihat secara visual.

PHASE 5: SKENARIO 2 - PELATIHAN DENGAN DATA AUGMENTATION
Detail Subphase:
5.1 Re-training Model:
    - Melakukan training menggunakan dataset yang telah dicampur dengan data augmentasi low-light.
    - Menggunakan hiperparameter yang sama dengan Skenario 1 agar perbandingan adil (fair).
5.2 Monitoring Overfitting:
    - Memantau grafik loss pada data training vs data validasi.

PHASE 6: ANALISIS ROBUSTNESS DAN EVALUASI AKHIR
Detail Subphase:
6.1 Pengujian Perbandingan:
    - Menguji kedua model (Baseline vs Augmented) pada dataset validasi yang sama (terutama subset low-light).
6.2 Ekstraksi Metrik:
    - Menghitung peningkatan nilai mAP (mean Average Precision).
    - Menganalisis hasil deteksi (Visual Inspection) seperti True Positives dan False Negatives di kondisi gelap.
6.3 Kesimpulan:
    - Menentukan efektivitas pendekatan data augmentation terhadap ketahanan (robustness) deteksi wajah.

PHASE 7: PENYUSUNAN LAPORAN DAN DOKUMENTASI
Detail Subphase:
7.1 Olah Data: Merapikan grafik dan tabel hasil eksperimen.
7.2 Dokumentasi Kode: Merapikan notebook dan script agar bisa direproduksi.
7.3 Finalisasi Draft Skripsi Bab III & IV.
